# 产品判断原则

本文档定义 autoccTest 的产品方向与取舍原则，帮助 Agent 在开发过程中做出一致的决策。

## 核心用户

- **测试工程师 / QA**：需要频繁执行回归验收测试，希望降低手工操作成本
- **开发团队**：希望在 CI/CD 流程中嵌入自动化验收测试

## 核心价值

- **降低门槛**：通过报文填写而非写代码的方式完成测试用例编写
- **通用接入**：被测系统无论使用什么协议和数据格式，都能通过适配器接入
- **可追溯**：每次验收结果可回溯、可对比，形成质量趋势

## 质量维度优先级

当前 0→1 阶段优先级从高到低：

1. **灵活性** — 报文模型和协议适配必须足够通用，避免后期重构
2. **可信度** — 断言结果准确可复现，不产生误导
3. **速度** — 尽快跑通端到端 MVP（报文编辑 → 执行 → 断言 → 报告）
4. **体验** — 早期不追求美观，但交互流程必须清晰
5. **成本** — 优先使用开源方案，避免供应商锁定

## 取舍原则

- 先做一个协议（HTTP/REST）的完整链路，再扩展到其他协议 —— 不要一开始就追求全协议覆盖
- 报文模型先支持 JSON，再扩展到 XML、Protobuf 等
- 先用单机调度验证核心流程，再考虑分布式执行
- 产品功能优先于运营能力（先能用，再好用）
